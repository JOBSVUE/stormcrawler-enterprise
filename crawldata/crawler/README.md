# StormCrawler Example

This is a complete, ready-to-run StormCrawler recipe for web crawling.

## Project Structure

```
crawler/
├── pom.xml                           # Maven configuration
├── crawler.flux                      # Flux topology definition
├── crawler-conf.yaml                 # Crawler configuration
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/example/crawler/
│   │   │       └── CrawlTopology.java  # Topology implementation
│   │   └── resources/
│   │       ├── parsefilters.json     # HTML parsing filters
│   │       ├── urlfilters.json       # URL filtering rules
│   │       └── seeds.txt             # Initial URLs to crawl
└── target/                          # Generated by Maven
```

## Building and Running

1. **Build the project**:

   ```bash
   cd crawler
   mvn clean package
   ```

2. **Run locally**:

   ```bash
   java -jar target/stormcrawler-example-1.0-SNAPSHOT.jar --local crawler.flux
   ```

   Or using Maven:

   ```bash
   mvn exec:java@run-local
   ```

3. **Submit to Storm cluster**:

   ```bash
   storm jar target/stormcrawler-example-1.0-SNAPSHOT.jar org.apache.storm.flux.Flux crawler.flux --remote
   ```

## Configuration

- Edit `seeds.txt` to add initial URLs
- Modify `urlfilters.json` to control which URLs to follow
- Update `parsefilters.json` to extract different content
- Adjust `crawler-conf.yaml` for crawler behavior settings

## Prerequisites

- Java 8+
- Apache Storm (for cluster deployment)
- Elasticsearch 7.x (for storage)

Note: This example uses Elasticsearch for storage. Make sure Elasticsearch is running at the configured address (default: localhost:9200).
