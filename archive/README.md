# Prerequisites

You need to install Apache Storm. The instructions on [setting up a Storm cluster](https://storm.apache.org/releases/2.5.0/Setting-up-a-Storm-cluster.html) should help. Alternatively, 
the [stormcrawler-docker](https://github.com/DigitalPebble/stormcrawler-docker) project contains resources for running Apache Storm on Docker. 

You also need to have an instance of URLFrontier running. See [the URLFrontier README](https://github.com/crawler-commons/url-frontier/tree/master/service); the easiest way is to use Docker, like so:

```
docker pull crawlercommons/url-frontier
docker run --rm --name frontier -p 7071:7071  crawlercommons/url-frontier
```

# Compilation

Generate an uberjar with

``` sh
mvn clean package
```

# URL injection

The next step is to inject URLs into URLFrontier, using the [client](https://github.com/crawler-commons/url-frontier/tree/master/client). Fortunately, it is added as a dependency to this project so all
you need to do is

``` sh
java -cp  target/crawler-1.0.jar crawlercommons.urlfrontier.client.Client PutURLs -f seeds.txt
```

where _seeds.txt_ is a file containing URLs to inject, with one URL per line.

# Running the crawl

You can now submit the topology using the storm command:

```sh
storm jar target/crawler-1.0.jar com.digitalpebble.CrawlTopology -- -conf crawler-conf.yaml
```

## Debugging Issues

If the spout shows 0 emitted tuples:

1. **Check Oracle database connection**:
   ```sql
   -- Connect to Oracle and verify data
   sqlplus c##mojtaba/bjnSY55l0g1IrzWY71Jg@//oracle-test:1521/XE
   
   -- Check total URLs
   SELECT COUNT(*) FROM crawl_queue;
   
   -- Check URLs by status
   SELECT status, COUNT(*) FROM crawl_queue GROUP BY status;
   
   -- View sample URLs
   SELECT url, status FROM crawl_queue WHERE ROWNUM <= 10;
   
   -- Reset all statuses to NEW for testing
   EXECUTE reset_all_statuses;
   ```

2. **Check Storm worker logs** for JDBC errors:
   ```bash
   docker logs supervisor
   docker logs nimbus
   
   # For more detailed logs from the crawler
   docker exec supervisor cat /logs/workers-artifacts/*/*/worker.log | grep -i oracle
   docker exec supervisor cat /logs/workers-artifacts/*/*/worker.log | grep -i jdbc
   ```

3. **Verify JDBC driver** is in the classpath:
   ```bash
   # Check if ojdbc8.jar exists
   ls -la /crawldata/crawler/lib/ojdbc8.jar
   
   # Verify it was properly installed in the Maven repo
   ls -la /crawldata/.repository/com/oracle/database/jdbc/ojdbc8/19.3.0.0/
   ```

4. **Check Elasticsearch indices**:
   ```bash
   # Verify content index exists
   curl -X GET "http://localhost:9200/content/_count"
   
   # Verify status index exists
   curl -X GET "http://localhost:9200/status/_count"
   ```

5. **Verify environment variables**:
   ```bash
   # Check if environment variables are properly set
   docker exec runner env | grep JDBC
   ```

If the issue persists, you may need to restart the topology after fixing the database:
```bash
docker-compose restart runner
```

To view detailed logs from the SimpleOracleSpout:
```bash
docker exec supervisor grep -i "SimpleOracleSpout" /logs/workers-artifacts/*/*/worker.log
```
